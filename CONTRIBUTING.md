# Contributing to Uni Event Hub

Thank you for your interest in contributing to Uni Event Hub! We welcome contributions from the community to help make this platform better for everyone.

By participating in this project, you agree to abide by our code of conduct and follow the guidelines outlined below.

## How to Contribute

### 1. Fork the Repository

Start by forking the repository to your own GitHub account.

### 2. Clone the Repository

Clone your forked repository to your local machine:

```bash
git clone https://github.com/YOUR_USERNAME/uni-event-hub.git
```

### 3. Create a Branch

Create a new branch for your feature or bug fix. Use a descriptive name:

```bash
git checkout -b feat/your-feature-name
# or
git checkout -b fix/your-bug-fix-name
```

### 4. Commit Your Changes

When committing your changes, please follow our commit message guidelines.

> [!IMPORTANT]
> **Commit Message Requirement:** Your commit message **must** clearly state the feature or change being implemented.
>
> **Atomic Commits:** Please use multiple commits if you are implementing multiple features or changes. Avoid bundling unrelated changes into a single commit. This helps us maintain a clean and understandable project history.

**Examples of good commit messages:**

- `feat: implement user registration and login`
- `fix: resolve issue with event date formatting`
- `docs: update API documentation for event creation`
- `refactor: optimize database queries for event search`

### 5. Push and Pull Request

Push your changes to your fork and create a Pull Request (PR) against the `main` branch of the original repository.

- Provide a clear description of the changes in your PR.
- Link any related issues.
- Ensure your code follows the project's coding standards.

## Coding Standards

- Follow the existing code style and patterns in the repository.
- Ensure your code is well-commented where necessary.
- Write clean, maintainable, and efficient code.

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub with a detailed description and steps to reproduce (for bugs).

---

We appreciate your contributions and look forward to building a great platform together!
